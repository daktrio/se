## 3. 요구사항 명세

### 3.1 CRC 카드 (Class-Responsibility-Collaborator)

| Class Name: **파일(File)** | ID: 01 |
|:---|:---|
| **Responsibilities (책임)** | **Collaborators (협력자)** |
| - 파일의 이름, 크기, 유형 등 메타데이터 관리 | - 데이터베이스(DB) |
| - 스토리지 내 실제 물리적 경로 정보 유지 | - 파일 스토리지 |
| - 버전 변경 시 히스토리 인덱스 생성 | - 버전 관리자 |

| Class Name: **공유(Share)** | ID: 02 |
|:---|:---|
| **Responsibilities (책임)** | **Collaborators (협력자)** |
| - 사용자별 접근 권한(조회/수정) 검증 | - 사용자(User) |
| - 공유 링크의 유효 기간 및 만료 여부 판단 | - 파일(File) |

### 3.2 동적 프로세스 상세 설명
1. **파일 업로드**: 사용자가 파일을 전송하면 시스템은 현재 사용자의 저장 용량을 확인한 후 스토리지에 저장하고 DB에 메타데이터를 기록한다.
2. **버전 생성**: 기존에 존재하는 파일명으로 덮어쓰기 업로드 발생 시, 시스템은 기존 데이터를 '이전 버전'으로 처리하고 새로운 데이터를 '최신 버전'으로 갱신한다.
3. **검색 실행**: 사용자가 검색어를 입력하면 시스템은 DB 내 인덱싱된 파일명을 실시간으로 필터링하여 결과를 반환한다.
