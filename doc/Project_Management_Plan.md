## 1. 서론

### 1.1 문서의 목적 및 범위
본 프로젝트 계획서는 인터넷 중고 책 거래 시스템에 대한 전반적인 내용을 정리하여 프로젝트를 진행하는 데 있어 각 세부 사항의 지표로 삼는 것을 목적으로 한다.

### 1.2 프로젝트 개요
사용자들이 소유한 중고 도서를 신뢰 기반으로 안전하게 거래할 수 있는 웹 기반 플랫폼을 개발한다.

### 1.3 프로젝트 목표
사용자가 원하는 중고 책을 실시간으로 검색하고 안전하게 결제하여 거래할 수 있는 서비스를 제공한다.

### 1.4 용어 정의
* **WBS**: 작업 분할 구조도, 전체 업무를 세분화한 도표
* **API**: 기능을 제어할 수 있게 만든 인터페이스

### 1.5 참조 문서
* 소프트웨어공학 강의자료 (Ch05. 프로젝트 관리)
* [KAU-SE]시스템정의서_260411_Doc-002

## 2. 개발 계획

### 2.1 개발 절차 모형
사용자의 요구사항을 빠르게 반영할 수 있는 Agile 방법 중 SCRUM 모델을 사용하며, 객체지향 방법론을 채택한다.

### 2.2 WBS(Work Breakdown Structure)
* **요구분석**: 계획서 검토, 기존 시스템 조사, 요구 분석 및 정의
* **설계**: 분석서 검토, 시스템 구조 설계, 상세 설계
* **코딩**: 기능의 구현, 서버 및 DB 구축, 각 부분 기능의 통합
* **테스트**: 전체적 테스트, 매뉴얼 작성, 발표 자료 작성

### 2.3 개발 내용
* **Sprint**: 2주 단위로 짧게 프로그램을 개발하면서 피드백 제공
* **Daily Scrum**: 매일 진행상황과 이슈 사항 공유

### 2.4 개발 일정
프로젝트 계획부터 종료까지 총 16주(4개월)의 일정을 준수한다.
